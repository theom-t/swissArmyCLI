# General Coding Rules

This project adheres to strict coding standards to maintain readability and maintainability.

## Comments and Documentation
*   **Concise Comments:** Code comments must be clear, concise, and explain *why* something is done, rather than *what* the code is doing (unless the code is unusually complex). Avoid redundant comments.
*   **Docstrings:** All functions, classes, and modules must have descriptive but concise docstrings.

## Assumptions
*   We are currently in the research stage. Do not make premature architectural or technology assumptions.
